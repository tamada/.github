# コントリビューションガイドライン

This document is the contributing guidelines written in Japanese.
English version is available at [CONTRIBUTING.md](CONTRIBUTING.md).

このプロジェクトへの貢献に興味を持っていただきありがとうございます！
私たちはすべての人からの貢献を歓迎します。

## 貢献方法

### Issue の報告

- 既に報告されていないか確認してください。
- 利用可能な場合は Issue テンプレートを使用してください。
- 明確な説明と再現手順を提供してください。
- 関連する環境の詳細を含めてください。

### プルリクエストの提出

1. **リポジトリをフォーク**し、`main` からブランチを作成する。
2. **変更を加える**際は明確で説明的なコミットメッセージを記述する。
3. **変更を十分にテストする。**
4. **必要に応じてドキュメントを更新する。**
5. **明確な説明とともにプルリクエストを提出する。**

### プルリクエストのガイドライン

- 既存のコードスタイルと規約に従ってください。
- 明確なコミットメッセージを記述してください。
- 変更は焦点を絞り、アトミックに保ってください。
- 新機能にはテストを追加してください。
- 提出前にすべてのテストが通ることを確認してください。

### コードスタイル

- プロジェクトのコーディング規約に従ってください。
- 意味のある変数名と関数名を使用してください。
- 複雑なロジックにはコメントを付けてください。
- コードをクリーンで読みやすく保ってください。

### コミットメッセージの形式

```
<type>: <subject>

<body>

<footer>
```

タイプ: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`

例:
```
feat: ユーザー認証機能を追加

JWT ベースの認証システムを実装
ログインとログアウト機能を含む

Closes #123
```

## 開発環境のセットアップ

1. リポジトリをクローンする。
2. 依存関係をインストールする。
3. フィーチャーブランチを作成する。
4. 変更を加える。
5. テストを実行する。
6. プルリクエストを提出する。

## 行動規範

このプロジェクトは[行動規範](CODE_OF_CONDUCT.ja.md)のもとで公開されています。
このプロジェクトに参加することで、その条項に従うことに同意したものとみなされます。

## 質問がありますか？

質問や懸念事項については、お気軽に Issue を開いてください。

## ライセンス

貢献することにより、あなたの貢献がプロジェクトと同じライセンスの下でライセンスされることに同意したものとみなされます。
